Highlight #1 Version Control

This topic is pretty relevant to me and it was refreshing to cover it in class to say the least. Version control is extremely essential in today's growing world of cloud collaboration. By having a main repository, you put all of your eggs in one basket, and it eliminates many risks like missing files that disable project workflow or file thefts. I personally use a cloud collaboration tool for music producing called Splice that has an amazing Version Control system.

Highlight #2 Remote vs. Local Repositories

Splice for an example, uses both remote and local repositories. The image displayed for this entry is the user interface for the desktop app, which shows the local repository. Once a music producer like myself finishes an edit on a song, the save file goes into a local repository (your computer). If the track has other collaborators, they will be notified of the change as the file transfers over to the remote repository, in this case, Splice's servers. Once the file is in the remote repository, the other collaborators can download it and continue to work on the new version of the song! Cloud collaboration is truly a fascinating thing!

Highlight #3 What is GIT? Why should we use it?

In sum, GIT is a free version control system that allows for the handling of projects and the flow of information in an efficient manner. Git is specifically useful because every single user has a complete copy of the repositories data to be stored locally which is great because the user can then work locally without the need to be connected to the network. It also prevents unintentional overwriting of files in the shared repository.
